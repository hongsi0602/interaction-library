# 롤백 가이드

작업 중 특정 시점으로 되돌리려면 Git을 사용합니다.

## 1. 체크포인트 만들기 (되돌릴 수 있는 시점 저장)

중요한 작업이 끝날 때마다 커밋하면, 나중에 그 시점으로 롤백할 수 있습니다.

```bash
cd interaction-library

# 변경 사항 확인
git status

# 전체 스테이징
git add -A

# 체크포인트 메시지와 함께 커밋
git commit -m "체크포인트: Confetti/CORE7 목록만 남김, 실미리보기 썸네일 적용"
```

원하는 메시지 예:
- `체크포인트: 사용처별 코드 탭 추가`
- `체크포인트: Confetti 무한반복 적용`
- `체크포인트: 상세 페이지 스타일 정리`

## 2. 마지막 커밋 시점으로 롤백 (작업 취소)

**아직 커밋하지 않은 변경만** 모두 버리고, **가장 최근 커밋 상태**로 되돌리기:

```bash
git checkout -- .
git clean -fd
```

- `git checkout -- .` : 추적 중인 파일을 마지막 커밋 내용으로 복구
- `git clean -fd` : 추적 안 하는 새 파일/폴더 삭제

## 3. 이전 커밋 시점으로 롤백

**과거 특정 커밋**으로 프로젝트 전체를 되돌리기:

```bash
# 커밋 이력 보기 (되돌릴 시점 확인)
git log --oneline

# 예: 5글자 이상 커밋 해시만 보기
git log --oneline -10
```

나온 해시(예: `a1b2c3d`)를 복사한 뒤:

```bash
# 해당 커밋 상태로 되돌리기 (이후 커밋은 남아 있음)
git checkout a1b2c3d -- .

# 또는 그 커밋 시점으로 완전히 이동 (이후 커밋은 사라짐)
git reset --hard a1b2c3d
```

- `git checkout <해시> -- .` : 그 커밋의 파일 내용만 현재 폴더에 가져옴. 이후에 다시 `git checkout main` 등으로 브랜치로 돌아올 수 있음.
- `git reset --hard <해시>` : HEAD를 그 커밋으로 옮기고, 그 이후 커밋은 제거됨. 신중히 사용.

## 4. 요약

| 목적 | 명령 |
|------|------|
| 체크포인트 저장 | `git add -A` → `git commit -m "메시지"` |
| 마지막 커밋으로 되돌리기 | `git checkout -- .` + `git clean -fd` |
| 이전 커밋으로 되돌리기 | `git log --oneline` 후 `git checkout <해시> -- .` 또는 `git reset --hard <해시>` |

이 저장소는 이미 **현재 상태**가 첫 커밋으로 남겨져 있으므로, 위처럼 커밋을 쌓아가면 언제든 그 시점으로 롤백할 수 있습니다.
